## Context

### Original Request
User requested features for v1.0.0 major release:
1. OpenTelemetry improvements
2. TLS improvements (certificate rotation)
3. Lifecycle cleanup (auto-cleanup after tasks)
4. Rust + Go build support
5. WAN networking support

### Research Findings

**OpenTelemetry gaps:**
- `handleFlutterBuild()` — NO span
- `handleUnityBuild()` — NO span
- `Worker.Build()` — NO span
- CLI commands — NO application spans
- `--tracing-headers` flag missing

**TLS gaps:**
- NO certificate rotation (certs loaded once at startup)
- NO cert expiration validation
- NO path existence check before loading

**Lifecycle cleanup gaps:**
- NO cleanup if process crashes before defer runs
- NO periodic cleanup of orphaned temp directories
- NO cache `.tmp` file cleanup on startup

**WAN gaps:**
- mDNS doesn't work across subnets
- No central registry for WAN workers
- No persistence — coordinator restart loses workers

**Rust/Go:**
- Rust: Full project archive pattern (like Flutter/Unity)
- Go: Full project archive with shared cache
- Proto already defines `BUILD_TYPE_RUST = 5`, `BUILD_TYPE_GO = 6`

### Definition of Done
- [ ] All builds pass: `go build ./...`
- [ ] All tests pass: `go test -race ./...`
- [ ] Lint passes: `golangci-lint run`
- [ ] `hgbuild rust build` distributes to workers
- [ ] `hgbuild go build` distributes to workers
- [ ] Workers auto-cleanup temp files on completion
- [ ] TLS certs hot-reload on SIGHUP
- [ ] WAN workers can connect across networks
- [ ] Dashboard shows WAN worker status
- [ ] All new features documented

### Must Have
- Lifecycle cleanup manager with periodic sweeps
- OTel spans for Flutter/Unity/Build handlers
- TLS certificate rotation via file watcher
- Rust executor for `cargo build`
- Go executor for `go build`
- WAN registry with persistent storage
- Hybrid discovery (mDNS + explicit coordinator)

### Must NOT Have (Guardrails)
- NO multi-coordinator HA (defer to v2.0)
- NO NAT traversal via STUN/TURN (manual port forward sufficient)
- NO distributed registry (etcd/consul) — central coordinator is enough
- NO Rust incremental compilation sharing (too complex)
- NO breaking changes to existing C++/Flutter/Unity builds
- NO changes to existing gRPC API (additive only)
