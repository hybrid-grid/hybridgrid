ROOT CAUSE:
-----------
**Test Infrastructure Bug: Script does NOT check exit codes**

Code Analysis (test/stress/run-test.sh):
-----------------------------------------

LINES 98-103 (THE BROKEN CODE):
```bash
DIST_START=$(date +%s.%N)
hgbuild --coordinator=${COORDINATOR} -v make -j8 2>&1 | tail -20
DIST_END=$(date +%s.%N)
DIST_TIME=$(echo "$DIST_END - $DIST_START" | bc)

print_success "Distributed build completed in ${DIST_TIME}s"
```

PROBLEM 1: No exit code capture
- Line 99 pipes stderr to stdout and pipes to `tail -20`
- Pipeline exit code = exit code of LAST command (tail), NOT hgbuild
- tail ALWAYS succeeds → pipeline always returns 0
- Script continues to line 103 and prints success message

PROBLEM 2: Exit code visible but not checked
- Log line 124-125 shows "Error: exit status 2"
- This comes from hgbuild's stderr (before tail truncation)
- Script never checks $? after the pipeline

PROBLEM 3: Divide-by-near-zero speedup
- DIST_TIME = 0.106s (time to fail, not compile)
- LOCAL_TIME = 635s (actual compilation)
- Line 106: SPEEDUP = 635 / 0.106 = 5978x (meaningless)

ACTUAL FAILURE:
---------------

From log lines 106-123 (make --help output), the actual error was:

```
hgbuild --coordinator=coordinator:9000 -v make -j8
```

This shows `make --help` instead of compilation output. Possible causes:

HYPOTHESIS A: hgbuild not in PATH in builder container
- Dockerfile.base:45 copies to /usr/local/bin/hgbuild
- /usr/local/bin should be in PATH by default
- UNLIKELY — would fail earlier in script (line 94 checks DISTRIBUTED_AVAILABLE)

HYPOTHESIS B: Flag parsing bug
- `hgbuild --coordinator=coordinator:9000 -v make -j8`
- If hgbuild incorrectly parses flags, it might pass `-v` to make
- make -v shows version, then exits
- POSSIBLE

HYPOTHESIS C: DisableFlagParsing issue
- hgbuild uses cobra.Command.DisableFlagParsing for wrapper commands
- If "make" subcommand doesn't have this, cobra might steal `-j8`
- POSSIBLE

HYPOTHESIS D: Preprocessing failed with exit 2
- hgbuild preprocessing step failed (OS mismatch?)
- Builder container is Linux (aarch64-unknown-linux-gnu per line 122)
- Local build succeeded, so sources are OK
- LIKELY — similar to Task 5 "no worker available" issue

RECOMMENDED FIX (for future work, out of E2E scope):

**Fix 1: Capture exit code correctly**

test/stress/run-test.sh lines 98-106:

BEFORE:
```bash
DIST_START=$(date +%s.%N)
hgbuild --coordinator=${COORDINATOR} -v make -j8 2>&1 | tail -20
DIST_END=$(date +%s.%N)
DIST_TIME=$(echo "$DIST_END - $DIST_START" | bc)

print_success "Distributed build completed in ${DIST_TIME}s"
```

AFTER:
```bash
DIST_START=$(date +%s.%N)
BUILD_LOG=$(mktemp)
if hgbuild --coordinator=${COORDINATOR} -v make -j8 > "$BUILD_LOG" 2>&1; then
    DIST_END=$(date +%s.%N)
    DIST_TIME=$(echo "$DIST_END - $DIST_START" | bc)
    print_success "Distributed build completed in ${DIST_TIME}s"
    tail -20 "$BUILD_LOG"
else
    DIST_END=$(date +%s.%N)
    DIST_TIME=$(echo "$DIST_END - $DIST_START" | bc)
    print_error "Distributed build FAILED in ${DIST_TIME}s"
    tail -50 "$BUILD_LOG"  # Show more context on failure
    rm -f "$BUILD_LOG"
    exit 1
fi
rm -f "$BUILD_LOG"
```
