## Cache Metrics Architecture Discovery

**Important Finding**: The coordinator does NOT have its own cache.

**Cache Flow**:
```
Client (hgbuild)
  └─> Client Cache (cache.Store) ← Instrumented here
      ├─> HIT: Return immediately
      └─> MISS: Send to Coordinator
          └─> Coordinator
              └─> Forwards to Worker
                  └─> Worker Cache (if implemented)
```

**Impact**:
- ✅ Cache instrumentation in `internal/cache/store.go` IS correct
- ✅ Cache metrics WILL appear at client metrics endpoints (not implemented yet)
- ✅ Cache metrics WILL appear at worker metrics endpoints (if workers expose metrics)
- ❌ Cache metrics will NOT appear at coordinator `/metrics` endpoint
- ℹ️  This is EXPECTED BEHAVIOR, not a bug

**Evidence**:
```bash
# Client logs show cache working:
[cache] main.c -> main.o (0.00s)  ← Cache hit on second compile

# Coordinator has no cache.Store usage:
grep -r "cache.Store\|cache.New" internal/coordinator/ cmd/hg-coord/
# (no results)
```
